# 代購作業台規格

## 目的

建立一個手機優先的 PWA 代購管理工具，讓使用者可在瀏覽器或手機主畫面中管理代購團、匯入賣貨便訂單 Excel、追蹤採購進度，並估算成本與獲利。

## 目標平台

- 桌面瀏覽器
- 手機瀏覽器
- 可安裝式 PWA

PWA 安裝與離線快取需透過 `http://localhost` 或正式 `https` 網址開啟。直接開啟 `index.html` 時需保留主要功能，但 service worker 不啟用。

## 使用情境

1. 使用者新增一個代購團並匯入 Excel。
2. 系統讀取 `非訂單匯入` 工作表。
3. 系統依商品名稱彙整商品、單價、訂購數量與訂單筆數。
4. 使用者輸入對客報價匯率與實際成本匯率。
5. 使用者逐項維護日幣原價與已代購數量。
6. 系統計算尚未代購數量、成本單價、單件獲利、總獲利與利潤率。
7. 後續同一團訂單變動時，使用者重新匯入 Excel 更新該團商品清單。

## 資料來源

- 檔案格式：Excel `.xlsx`
- 指定工作表：`非訂單匯入`
- 不使用 `訂單匯入` 工作表

## Excel 欄位對應

| App 欄位 | Excel 來源欄位 | 說明 |
| --- | --- | --- |
| 商品名稱 | `商品名稱 (品名/規格)` | 用來彙整代購項目 |
| 訂單單價 | `單價` | 客人下單的台幣售價 |
| 訂購數量 | `數量` | 同商品加總 |
| 訂單編號 | `訂單編號` | 用於統計商品來自幾筆訂單 |

## 代購團資料

每個代購團包含：

- 代購團名稱，可由匯入檔名建立預設值，後續可手動修改
- 建立時間
- 最後更新時間
- 最後匯入的檔案名稱
- 對客報價匯率
- 實際成本匯率
- 商品彙整清單
- 商品手動設定資料
- 最近一次匯入摘要

## 匯入與更新規則

新增代購團時：
- 建立新的獨立代購團。
- 商品依名稱彙整。
- 匯入摘要顯示新增商品數。

更新既有代購團時：
- 重新讀取 `非訂單匯入`。
- 重新依商品名稱彙整商品。
- 更新商品單價、訂購數量與訂單筆數。
- 新商品加入清單。
- 新匯入中不存在的商品從目前清單移除。
- 不覆蓋既有商品的日幣原價。
- 不覆蓋既有商品的已代購數量。
- 不覆蓋該團的對客報價匯率與實際成本匯率。
- 不覆蓋使用者已修改的代購團名稱。
- 匯入摘要需顯示新增、變更、移除與保留設定數量。

## 匯率設定

系統提供兩種匯率：

- 對客報價匯率：用於從台幣售價回推商品日幣原價。
- 實際成本匯率：用於計算採購成本與獲利。

兩者不可共用同一欄位。

## 商品資料

每個商品項目包含：

- 商品名稱
- 訂單單價
- 訂購總數量
- 已代購數量
- 尚未代購數量
- 商品日幣原價
- 成本單價
- 單件獲利
- 總獲利
- 利潤率
- 訂單筆數
- 狀態：待代購、已完成、缺成本、低利潤

## 計算公式

商品日幣原價回推：

```text
商品日幣原價 = 訂單單價 / 對客報價匯率
```

商品工作表需提供「全部回推日幣」操作，批次將尚未設定日幣原價的商品依公式補上。此操作不覆蓋已手動設定的日幣原價。

成本單價：

```text
成本單價 = 商品日幣原價 × 實際成本匯率
```

單件獲利：

```text
單件獲利 = 訂單單價 - 成本單價
```

總獲利：

```text
總獲利 = 單件獲利 × 訂購總數量
```

尚未代購數量：

```text
尚未代購數量 = 訂購總數量 - 已代購數量
```

利潤率：

```text
利潤率 = 總獲利 / 預估營收
```

## 介面規格

整體定位為「代購作業台」，不是行銷頁或單純資料表。

主要區域：
- 頂部命令列：顯示工具名稱、保存狀態、安裝、備份與新增代購團。
- 左側代購團清單：支援搜尋與狀態篩選。
- 目前代購團作業區：顯示可編輯團名、匯入摘要、成本設定、彙總 KPI、商品採購工作表。
- 商品詳情抽屜：手機與桌面都可用於單項商品編輯。

手機版：
- 以卡片式商品清單為主要操作方式。
- 商品詳情使用抽屜呈現。
- 重要按鈕與輸入欄位高度至少 44px。
- 避免讓使用者在手機上依賴寬表格操作。
- 匯率欄位輸入時不可每次按鍵重寫 input value，避免輸入 `.` 或 `0.` 時游標跳動。

桌面版：
- 保留高密度表格。
- 表格需具備 sticky 表頭、sticky 商品欄、斑馬紋與 hover 狀態。
- 表格上方提供快速篩選。

可存取性：
- 狀態文字使用 live region。
- 按鈕與輸入欄位需有明顯 focus 樣式。
- 狀態不可只依賴顏色，需有文字標籤。
- 支援 `prefers-reduced-motion`。
- CSS 需保留 `[hidden] { display: none !important; }`，避免元件 display 樣式覆蓋 HTML hidden 狀態。

## PWA 規格

- 提供 `manifest.webmanifest`。
- 提供 `service-worker.js` 快取 app shell。
- 提供 SVG app icon。
- Service worker 僅在 `http` 或 `https` 環境註冊。
- 直接開啟本機檔案時，PWA 安裝與離線快取不啟用。
- 調整 app shell 檔案後需更新 service worker cache 版本，避免瀏覽器沿用舊 CSS 或 JS。

## 資料保存

目前第一階段使用 `localStorage`。

儲存內容：
- 代購團列表
- 各代購團匯率
- 各代購團商品彙整資料
- 各商品日幣原價
- 各商品已代購數量
- 最近一次匯入摘要

需支援從舊版 `purchase-tracker-state-v2` 讀取資料。

## 備份與匯出

- 全部資料可匯出 JSON 備份。
- 單一代購團可匯出 JSON 清單。
- 後續可擴充 Excel / CSV 匯出。

## 後端擴充需求

個人單機使用第一版不需要後端。

若未來需要以下能力，需新增後端：
- 跨手機、跨瀏覽器同步
- 使用者登入
- 多人維護同一代購團
- 匯入歷史與回復
- 自動備份
- 雲端附件或商品備註
- 從外部 API 抓日幣匯率

建議 API：

```text
GET    /api/groups
POST   /api/groups
GET    /api/groups/:id
PATCH  /api/groups/:id
POST   /api/groups/:id/imports
PATCH  /api/groups/:id/items/:itemId
GET    /api/groups/:id/export
```

建議資料表：

```text
groups
imports
items
item_overrides
```

## 第一版不包含

- 後端服務
- 使用者登入
- 雲端資料庫
- 多裝置同步
- 自動取得日幣匯率
- Excel / CSV 匯出

## 維護流程

後續新增或調整功能時，採用以下流程：

1. 先分析現有資料、程式與需求背景。
2. 判斷需求中已明確與不明確的部分。
3. 若不明確處會影響資料模型、流程、計算公式或使用方式，先詢問使用者。
4. 確認後整理規格與實作計畫。
5. 再開始小範圍修改。
6. 完成後執行可行的檢查或測試。
7. 同步更新 `SPEC.md` 與 `README.md`。
